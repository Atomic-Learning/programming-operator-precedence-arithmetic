When an expression contains more than one arithmetic operator, the operators are evaluated in a specific order called operator precedence. This order is almost universal across programming languages and matches standard mathematical conventions:

  1. Exponentiation
  2. Multiplication and division (and related operators like modulo)
  3. Addition and subtraction

For example, in the expression 2 + 3 * 4, multiplication is performed before addition, so the result is 14 (not 20).

Parentheses

Parentheses ( ) can be used to override the default precedence and make the order of operations explicit. When multiple pairs of parentheses are nested, operations within the innermost parentheses are evaluated first.

2 + 3 * 4     → 14
(2 + 3) * 4   → 20
2 ** 3 * 2    → 16
2 ** (3 * 2)  → 64

Associativity: Order for Same-Tier Operators

When an expression contains multiple operators of the same precedence (for example, several additions or multiplications in a row), the order in which they are evaluated is called associativity:

  • Left-to-right (left-associative): Most arithmetic operators (such as addition, subtraction, multiplication, division) are evaluated from left to right.
  • Right-to-left (right-associative): Exponentiation is usually evaluated from right to left.

For example:

10 - 3 - 2   →  (10 - 3) - 2   → 5
2 ** 3 ** 2  →  2 ** (3 ** 2)  → 512
